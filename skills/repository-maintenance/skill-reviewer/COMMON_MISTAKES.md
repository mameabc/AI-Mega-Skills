# Skill Reviewer Common Mistakes

- Approving because the Markdown is polished while the workflow is untestable.
- Checking only the changed file and missing contradictory companion documents.
- Treating a similar name as proof of duplication without comparing scope.
- Accepting invented references or validation results.
- Reporting preferences as blockers without explaining impact.
- Approving with stale registry or README output.
- Rewriting the contributor's work instead of documenting findings.
