# Skill Reviewer Troubleshooting

## Scope Is Ambiguous

Pause the decision and ask the proposal owner to define the primary problem, triggers, non-goals, and expected output.

## Possible Duplicate

Compare inputs, outputs, decision boundaries, and validation behavior with related Skills. Recommend extension, refactor, or a new Skill only after the comparison.

## Automated Check Fails

Report the exact command and failure, treat it as blocking, and request a correction before evaluating approval.

## Evidence Is Missing

Mark the claim unverified. Request a real example, official reference, or command output rather than inferring correctness.

## Review Template Does Not Fit

Preserve the required decision and findings, then explain the missing review dimension to maintainers before changing the template.
