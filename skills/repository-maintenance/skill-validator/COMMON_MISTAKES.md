# Skill Validator Common Mistakes

- Running validation from a directory other than the repository root.
- Reporting a previous successful run as current evidence.
- Running only the validator and missing stale generated outputs.
- Fixing generated files manually instead of correcting the source.
- Treating a green script as proof that the Skill is useful or non-duplicative.
- Omitting failures from the final report.
- Modifying unrelated files while trying to repair a check.
