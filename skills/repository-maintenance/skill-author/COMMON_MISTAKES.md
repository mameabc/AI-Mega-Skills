# Skill Author Common Mistakes

- Creating a new Skill when an existing one should be improved.
- Leaving template tokens or generic prose in a generated directory.
- Describing prompts instead of teaching decisions and verification.
- Using invented examples, owners, references, or test results.
- Omitting the not-applicable boundary and allowing unsafe scope expansion.
- Editing generated registry or README content by hand.
- Treating repository validation as a substitute for human review.
