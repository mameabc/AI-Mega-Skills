# Skill Author Best Practices

- Start with a concrete user problem and observable output.
- Search the registry and Skill directories before naming a new Skill.
- Write the decision boundaries before the step-by-step workflow.
- Make examples executable or inspectable with the repository's actual tools.
- Explain why a rule exists when it prevents a likely failure.
- Keep generated outputs deterministic and separate from authored content.
- Report validation evidence, assumptions, and blocked decisions explicitly.
