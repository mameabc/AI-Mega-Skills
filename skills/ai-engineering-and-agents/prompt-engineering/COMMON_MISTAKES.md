# Prompt Engineering Common Mistakes

- Asking for “the best code” without naming behavior, constraints, or tests.
- Using a senior persona to replace missing context and acceptance criteria.
- Packing the prompt with irrelevant context that hides the authoritative files.
- Requiring a long explanation instead of an inspectable artifact and evidence.
- Assuming every coding agent exposes the same roles, tools, paths, or permissions.
- Telling the agent to continue when requirements conflict or authority is missing.
- Comparing prompt versions with different repositories, models, tools, or tasks.
- Treating one successful run as proof of prompt quality.
- Including credentials, private source, or real personal data in examples.
- Copying provider documentation or research text instead of linking and summarizing it.
