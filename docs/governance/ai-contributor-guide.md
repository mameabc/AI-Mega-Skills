# AI Contributor Guide

AI contributors must behave like maintainers: preserve existing work, make small changes, validate before claiming completion, and never invent project facts.

## Operating Rules

- Work only inside the repository checkout.
- Create a feature branch before changing files.
- Do not create skills unless one specific skill is requested.
- Do not rewrite core docs unless the task explicitly asks for it.
- Preserve user work and unrelated changes.
- Prefer improving existing files over adding overlapping documents.
- Use official documentation before third-party sources.
- Stop and ask when a requirement is ambiguous or missing.

## Required Workflow

1. Inspect current files and git status.
2. Confirm the change scope.
3. Make the smallest maintainable change.
4. Run repository validation.
5. Commit only after validation passes.
6. Push the branch when authentication is available.
7. Report changed files, validation, branch, and push status.

## Prohibited Behavior

- Bulk-generating skills.
- Creating duplicate working copies.
- Adding placeholder text.
- Claiming tests passed without running them.
- Fabricating pushes, links, citations, or review results.
- Mixing unrelated changes in one commit.
