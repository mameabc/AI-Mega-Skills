# Maintainers and Governance

## Roles

- **Maintainers:** Own repository-wide standards, releases, CI, and final merge authority.
- **Domain stewards:** Own quality and roadmap for a specific domain.
- **Reviewers:** Review pull requests against documented standards.
- **Contributors:** Propose and implement skills, docs, and tooling.

## Decision Rules

- Repository structure and schema changes require maintainer approval.
- Domain taxonomy changes require maintainer approval and domain steward review.
- New skills require at least one domain-aware review.
- Security-sensitive skills require maintainer review.

## Domain Steward Responsibilities

- Maintain the domain backlog.
- Prevent duplicate or low-value skills.
- Review examples and acceptance criteria.
- Track deprecations and replacements.
- Keep domain documentation current.

## Quality Escalation

If reviewers disagree, maintainers decide based on:

- User value.
- Safety.
- Maintainability.
- Testability.
- Fit with repository standards.
