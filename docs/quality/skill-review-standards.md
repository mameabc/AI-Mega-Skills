# Skill Review Standards

Skill review protects the library from duplication, vague guidance, unsafe workflows, and unmaintainable content.

## Review Questions

- Is the problem real and clearly scoped?
- Is the skill meaningfully different from existing skills?
- Does it improve agent reasoning or execution?
- Are required inputs and outputs explicit?
- Are examples realistic?
- Are best practices and common mistakes included?
- Is troubleshooting practical?
- Are references official or clearly justified?
- Does validation prove the skill works?
- Are README, registry, and category index updates included?

## Required Outcomes

Approve only when the skill passes the quality gate in [Skill Quality Standard](skill-quality-standard.md).

Request changes when the skill is useful but incomplete.

Reject when the skill is duplicative, unsafe, untestable, generic, or based on fabricated information.
