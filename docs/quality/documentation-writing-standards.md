# Documentation Writing Standards

Documentation should be concise, accurate, durable, and easy to review.

## Standards

- State the rule before explaining the reason.
- Use direct language.
- Prefer short sections over long essays.
- Link to the source of truth instead of repeating policy.
- Use examples only when they are real and useful.
- Avoid filler, marketing copy, and vague promises.
- Keep terminology consistent with repository naming conventions.

## Required Checks

- No placeholder text.
- No duplicated policy blocks.
- No broken internal links.
- No claims that depend on unavailable tools or private context.
- No generated examples presented as real usage.
